아파치 톰켓 = 아파치 + 톰켓 두개가 합쳐진거

아파치는 공유할 자원을 모아 둔 폴더다.

Server.xml – 8080 (포트설정)
Context.xml – DB연결
Web.xml – 필터 (Ip 차단 같은거)

ㄴ> 스프링에서는 application.properties 여기서 한번에 설정 가능

어파치는 reader로 읽어서 요청된 파일의 내용을 클라에게 응답함.

요청과 응답을 같이 하지만 스레드는 1개다 .
요청이 오면 응답을 하는 순서가 있어서 1개라는 뜻. 
BW 와 BR은 하나의 스레드로 묶여있다 ( 반이중 )

요청 request, 응답 responce 둘이 동기적으로 일어남.
요청이 있어야 응답을하고 응답을 받아야 요청을 하니까

톰캣은 브라우저가 이해할수 없는 언어 ( JAVA 관련 파일 ) 을 전달할때 필요하다.

톰캣은 java를 컴파일 해서 class로 만든 후에 실행 - > java파일의 결과값을 아파치한테 다시 돌려줌 
-> 데이터를 받으면 클라이언트한테 응답해줌.

간단히 말하면 아파치는 서버, 톰켓은 컴파일러다.
