통신 
 - 간단하게 0 과 1로 주고받는거
 - 선이 3가닥 있으면 6bit ( 경우의수 6개 , 물리적인 선 갯수가 많아 통신이 빠름)

 - 선 하나로 주고받는거 (1> 0> 1 이렇게 보내는 방식 )

 bit 통신보다 byte 통신이 효율적임 

 bit > 물 온도 체크할때 30도 아래면 0 위면 1 이런식으로 받는 경우는 bit로 할수 있음
  - > 0 받으면 온도 낮음 으로 컨버팅 해야함 - > 받을때 부터 온도 낮음으로 받으면 컨버팅 할 필요가 없음

 byte로 할시 8byte를 받을때 까지 모았다가 한번에 보냄 > 영어 한문자를 받아낼수 있어서 bit 통신보다 효율적 ( 컨버팅 안해도 됨 ) 

사진처럼 통신하려면 어려움 
 - 자바코드를 자바스크립트로 보내면 자바스크립트는 언어가 달라 재대로 전송이 안됨
 - 하지만 name = "강아지" 같이 컨버팅 해서 보내면 이해 가능.

=> 서로 다른 언어로 통신할때는 공용어가 필요함 

전송해야할 코드 
Class Dog{
 int age=10;
 String name = “강아지“;
}

공용어 종류 
 1.XML
<Dog>
 <age>10</age>
 <name>강아지</name>	
</Dog>

처럼 바꿈 - > 비효율적임 ( 파일의 크기가 켜저서 )

 2.JSON ( 자바스크립트오브젝터)
js 로 생성 

var Dog ={
 age : 10;
 name : "강아지"
}

{ "age":10,"name":"강아지"} 

class 명은 개발자 마음대로 적을 수 있으니 중요하지 않아 빼고 내부값만 씀 ( 최대한 용량을 줄이기 위해 )

모든 언어가 다 JSON을 다 들고 있음 ( 공용어 이기 때문 )
