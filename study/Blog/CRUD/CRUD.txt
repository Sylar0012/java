커넥션 성공시 세션이 만들어짐.

세션은 접속자의 정보를 저장해놓는 공간이라 보면 됨.

세션은 고유한 하나의 데이터임
서버는 세션컬렉터에 세션의 정보를 저장해놓고 요청시 세션컬렉터에 세션이 있나 없나 확인한다.

conn은 키 라고 볼 수 있다.

크롬으로 접속시 항상 서버에 연결해야한다.
직접적으로 접근하면 데이터를 수정및 삭제가 용이하게 이뤄지기 때문에

크롬으로 정보를 요청하여 서버에 접근시 DAO를 거쳐서 하게 만든다

DAO에 는 회원 목록() 회원한명정보()  회원 삭제() 회원 수정() 밖에 없다고 가정하면

사용자에게 해당하는 4개의 작업을 제외한 모든걸 막아버려 안전하게 만든 프로그램 이다.

즉 DAO에는 DB의 데이터를 안전하게 보호하기 위해 제약을 걸어놓은 통로이다.

DAO는 DB의 테이블 갯수만큼 만들어야 한다.


최초에는 GET밖에 없었다 ( select )
1.0에서 Post가 3가지를 담당했다가
1.1때 나눠짐 Put(수정), Delete(삭제), Post(추가)

4가지기능을 CRUD 라고 부름

CRUD = ( C = insert, R = Select, U = Update, D = delete )
 -> 기본적인 데이터 처리 기능을 묶어서 부르는 말.
